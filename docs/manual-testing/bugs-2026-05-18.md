## Flujos que funcionan correctamente

- Registro, login, logout y redirección de rutas protegidas (`/projects` → `/login?from=…`).
- Validación de formularios: proyecto sin nombre, reunión sin título, contraseñas distintas en registro, email duplicado, credenciales inválidas en login.

- Crear reunión y ver detalle; diálogo de confirmación al eliminar proyecto.
- Proxy API del frontend (`/api/v1/*` → backend) responde 200/201 en los flujos probados.

### BUG-015 — Banner “Permission denied” bloquea clics en la sala de videollamada

| Campo | Valor |
| --- | --- |
| **Severidad** | Alta |
| **Área** | Video call |
| **Ruta** | `/projects/:id/meetings/:meetingId/room` |
| **Archivo** | `features/video-call/VideoCallRoom.tsx` |

**Descripción:** Si el navegador deniega cámara/micrófono (común en automatización o sin permisos), aparece un banner rojo **“Permission denied”** dentro del área de contenido. Ese bloque intercepta eventos de puntero e impide usar el sidebar (p. ej. Dashboard) mientras la sala sigue abierta.

**Pasos para reproducir:**
1. Crear reunión → **Unirse a la llamada**
2. Denegar permisos de medios (o usar Playwright sin `--grant-permissions`)
3. Intentar clic en **Dashboard** en el sidebar

**Resultado esperado:** Banner no bloqueante, o sala a pantalla completa sin sidebar interactivo detrás.

**Resultado actual:** `playwright-cli click` falla: el div del error “subtree intercepts pointer events”.

### BUG-017 — Minuta no generada tras finalizar reunión

| Campo | Valor |
| --- | --- |
| **Severidad** | Alta |
| **Área** | Reuniones / IA |
| **Rutas** | `.../room`, `.../minutes` |

**Descripción:** Tras pulsar **Terminar**, la UI muestra “Procesando audio y generando minutas…”. Al ir a la página de minuta (`/minutes`), el mensaje es **“Aún no hay minuta para esta reunión.”** No hubo redirección automática ni contenido generado en la ventana de prueba.

**Pasos para reproducir:**
1. Crear reunión → unirse a la llamada → **Terminar**
2. Esperar overlay de procesamiento
3. Navegar a `.../meetings/:id/minutes`

**Resultado esperado:** Minuta generada o error visible con reintento.

**Resultado actual:** Pantalla vacía de minuta.

**Notas:** Probable dependencia del servicio AI (`task_manager_ai_back`, puerto 8000). En sesiones previas el backend AI fallaba por dependencia `aiofiles` no instalada. Verificar que el pipeline de subida + transcripción esté operativo.

### BUG-018 — Usuario puede quedar atrapado en overlay de procesamiento

| Campo | Valor |
| --- | --- |
| **Severidad** | Media |
| **Área** | Video call |
| **Archivo** | `VideoCallRoom.tsx` (`leaveStage === "processing"`) |

**Descripción:** Tras **Terminar**, si el procesamiento no completa (fallo de API/IA), la UI sustituye el grid por el spinner “Procesando audio…” sin botón para salir o volver al proyecto. La navegación lateral puede quedar bloqueada (véase BUG-015).

**Pasos para reproducir:**
1. Finalizar llamada con AI backend caído o sin audio grabado
2. Intentar salir sin esperar timeout

**Resultado esperado:** Botón “Volver al proyecto”, manejo de `onProcessingFailed`, o timeout con mensaje.

**Resultado actual:** Overlay persistente; en prueba manual el clic en sidebar falló por el banner de permisos.
